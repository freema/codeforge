# Phase 13: Enhanced PR Messages

## Cíl

Generovat **bohatší, strukturované PR popisy** s conventional commits, strukturovanými sekcemi, a kontextem z tasku.

## Závislosti

- Žádné přímé závislosti (malý scope, rychlý win)

## Současný stav

`internal/cli/analyzer.go` generuje základní PR metadata:

```go
type AnalysisResult struct {
    BranchSlug  string  // "task-{short_id}"
    PRTitle     string  // "CodeForge: {truncated_prompt}..."
    Description string  // "Automated changes by CodeForge."
}
```

To je minimální. Chceme:
- Strukturované PR popisy
- Conventional commits format
- Sekce: Summary, Changes, Testing, Context
- Review result (pokud existuje, Phase 12)
- Task metadata

## Implementace

### Commit Message Classification

```go
// internal/pr/classifier.go

type CommitType string

const (
    CommitFeat     CommitType = "feat"
    CommitFix      CommitType = "fix"
    CommitRefactor CommitType = "refactor"
    CommitDocs     CommitType = "docs"
    CommitTest     CommitType = "test"
    CommitChore    CommitType = "chore"
    CommitPerf     CommitType = "perf"
    CommitStyle    CommitType = "style"
)

type CommitClassification struct {
    Type    CommitType
    Scope   string  // extrahováno z cesty souborů nebo promptu
    Subject string  // krátký popis
}

func Classify(prompt string, changes *ChangesSummary, result string) *CommitClassification {
    // Heuristika:
    // - "fix", "bug", "repair", "resolve" → fix
    // - "add", "implement", "create" → feat
    // - "refactor", "simplify", "restructure" → refactor
    // - "test", "spec" → test
    // - "doc", "readme", "comment" → docs
    //
    // Scope: extrahuj z dominantního adresáře v changes
    // Subject: prvních 50 znaků promptu, upraveno
}
```

## Testovací strategie

### Classifier testy (`internal/pr/classifier_test.go`)

Nejdůležitější testy — musí pokrýt širokou škálu promptů:

```go
func TestClassify_CommitType(t *testing.T) {
    tests := []struct {
        name   string
        prompt string
        want   CommitType
    }{
        {"fix keyword", "Fix the login bug", CommitFix},
        {"fix keyword cz", "Oprav bug v přihlášení", CommitFix},
        {"resolve keyword", "Resolve race condition in workers", CommitFix},
        {"add feature", "Add user authentication", CommitFeat},
        {"implement feature", "Implement dark mode", CommitFeat},
        {"create feature", "Create new API endpoint", CommitFeat},
        {"refactor keyword", "Refactor executor pipeline", CommitRefactor},
        {"simplify keyword", "Simplify error handling", CommitRefactor},
        {"test keyword", "Add unit tests for auth", CommitTest},
        {"docs keyword", "Update README with API docs", CommitDocs},
        {"ambiguous - default feat", "Change the color of the button", CommitFeat},
    }
    for _, tt := range tests {
        t.Run(tt.name, func(t *testing.T) {
            got := Classify(tt.prompt, nil, "")
            if got.Type != tt.want {
                t.Errorf("Classify(%q).Type = %q, want %q", tt.prompt, got.Type, tt.want)
            }
        })
    }
}
```

- [ ] `TestClassify_CommitType` — table-driven: prompt → commit type (20+ případů)
- [ ] `TestClassify_Scope` — extrakce scope z dominantního adresáře v changes
- [ ] `TestClassify_Scope_NoChanges` — žádné changes → scope z promptu
- [ ] `TestClassify_Subject` — krátký, imperativní popis (max 50 znaků)
- [ ] `TestClassify_Subject_Truncation` — dlouhý prompt → oříznutý subject
